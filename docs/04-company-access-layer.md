# Company Access Layer

## Цель

На этом этапе добавлен минимальный слой доступа к компаниям.

Мы не делаем полноценное управление компаниями и участниками. Цель этапа — подготовить основу для API-токенов, импортов и экспортов.

## Роуты

| Роут | Назначение |
|---|---|
| `/companies` | Список компаний текущего пользователя |
| `/companies/{slug}` | Страница компании |

## Permissions

```text
COMPANY_VIEW
COMPANY_MANAGE
COMPANY_MEMBERS
COMPANY_API_TOKENS
```

## Логика доступа

| Пользователь | Доступ |
|---|---|
| ROLE_ADMIN | Всё |
| owner внутри компании | Всё по компании |
| admin внутри компании | Всё по компании |
| member внутри компании | Только просмотр |
| пользователь без membership | Нет доступа |

## Почему нужен Voter

Роль `ROLE_USER` говорит только то, что пользователь вошёл в систему.

Но она не говорит:

- состоит ли пользователь в конкретной компании;
- какую роль он имеет внутри компании;
- можно ли ему управлять API-токенами;
- можно ли ему видеть импорты и экспорты компании.

Поэтому логика доступа к компании вынесена в `CompanyVoter`.
