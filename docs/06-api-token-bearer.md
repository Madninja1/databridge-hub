# API Token / Bearer Token

## Цель

На этом этапе добавлена stateless API-аутентификация через Bearer token.

Пример запроса:

```http
Authorization: Bearer dtb_xxx
```

## Что реализовано

- сущность `ApiToken`;
- хранение только хэша токена;
- выпуск токена;
- отзыв токена;
- срок действия токена;
- `last_used_at`;
- Symfony Access Token Authenticator;
- JSON-ответы для `401`;
- консольная команда для первого токена.

## Почему opaque token, а не JWT

На этом этапе используется opaque token — случайная строка.

Плюсы:

- легко отозвать;
- легко хранить срок действия;
- легко инвалидировать;
- сервер полностью контролирует состояние токена.

JWT будет отдельным этапом.

## Почему не храним plain token

Plain token показывается только один раз при создании.

В БД хранится:

```text
hash_hmac('sha256', plainToken, API_TOKEN_HASH_SECRET)
```

Это снижает риск при утечке БД.
